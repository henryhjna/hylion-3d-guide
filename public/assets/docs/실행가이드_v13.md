> **듀얼 트랙 구조**
> - **Track A (상체):** d1(리드) + e1 + e2(부분)
> - **Track B (하체):** d2(리드) + d3 + e2(부분)
> - **Week 8 합류** -> 상하체 결합 -> 풀 시나리오 -> Week 10 발표

> **멤버 정체성**
> - d1: 하드웨어, 외장 오너 (A 리드): 상체 HW, 토르소, SO-ARM, 머리 제작, 외장 마감, 상하체 결합
> - d2: 보행 시스템 오너 (B 리드): 액추에이터~보행 E2E, 전원, 안전, BHL lowlevel, NUC 배포
> - d3: AI 인프라 오너 (A+B): 설계, 판단 + DGX 학습 실행 (SmolVLA 아키텍처, Walking RL 환경, 보상함수)
> - e1: 배포, 통합 오너 (A 리드): Orin, LeRobot 배포, 상태 머신, FETCH, 대화 파이프라인, 서바이벌, 최종 통합
> - e2: 캐릭터, 검증, 외장 오너 (A+B): 입 서보 제어, 감정 표현, 평가, sim-to-real 분석, 바디 외장, 키워드 사전

> **v12 -> v13 주요 변경:** ESP32/MOSFET 삭제, ROS2 삭제(plain UDP), 목 서보/시선 추적 삭제, 모터 12개(M6C12 x8+5010 x4), CAN x2, 배터리 2개(6S+4S), Piper TTS local, SmolVLA LeRobot PyTorch, Walking policy ONNX Runtime
